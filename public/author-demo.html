Welcome To CRSG Author Module

Game Description

You wrote a module for a project and opened a pull request. However, there were some problems with your code. CRSG reviewed your code and gave you some feedbacks. You must solve the problems and commit again until you solve all the problems.

Game Components

1) Editors

Left-hand side editor shows your initial code. It is read-only.

Right hand side editor shows your latest code. You should edit it and fix the errors in the original code.

Please don't change variable/function/class names or signatures because this will break the scoring system.

When you think you solved all of the problems in the code, click commit changes to get a new review.

You can resize all of the modules as you wish.

2) Reviews

You can see review comments in this tab. Beware! Some reviews may be incorrect because reviewers can make mistakes as well. You should reject a review if you think it doesn't make sense. You can click the show hint button to see a hint about a review comment as well.

3) Error Output

If your code has a compile-time or runtime error, it is shown in the error output.

4) Show Solutions

You can click show solutions to see the solutions. Note that you will not get any more points after you see the solutions.

5) Score System

Each review comment's worth are equal. You lose 20% score from each unsolved review comment when you commit changes.

You have 2 minutes for each review comment. (You can allocate your total time differently if you wish). If you run out of time, you will lose 1% of score per minute for reamining review comments.

The topmost penalty indicator shows much penalty you will get if you commit now.
